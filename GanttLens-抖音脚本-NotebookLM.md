# GanttLens 抖音 Demo 视频脚本

## 一、作品信息

- **作品名称**：GanttLens — 智能甘特图排期工具
- **核心定位**：让项目排期回归思考本身
- **视频时长**：55 秒
- **拍摄风格**：口播 + 录屏演示（可出镜可不出镜）
- **画面比例**：9:16 竖屏
- **发布平台**：抖音
- **活动话题**：#初赛Demo盲盒

## 二、目标受众

- 项目经理、产品经理、弱电/IT 项目工程师
- 经常使用甘特图但被"牵一发动全身"折磨的人
- 对 AI 辅助排期工具感兴趣的技术从业者

## 三、核心卖点（按冲击力排序）

1. **拖拽级联**：拖一个任务，总工期自动锁死，阶段条带同步联动——告别手动改下游
2. **文件夹扫描自动成图**：扫描项目文件夹，文件名里的日期直接吃进去，甘特图自动生成
3. **实时同步**：文件一改，甘特图实时跟着变（chokidar 监听）
4. **代号清晰**：项目代号列 M-2026 / DC-2026 / OFC-2026 一目了然
5. **悬停精准**：悬停只高亮当前项目，不串扰

## 四、钩子策略

- **选定钩子**：A 痛点型 ——「改一个日期，全盘崩？」
- **选择依据**：反差强，命中项目经理真痛点，与活动"Demo 能跑通、能体验到价值"调性一致
- **备选钩子**：
  - B 效果型：直接上拖拽级联画面 +「这一拖，省你两小时」
  - C 疑问型：「甘特图还能这么玩？」

## 七、录屏操作清单（按镜号准备素材）

1. **镜2 痛点素材**：找旧版/竞品甘特图，演示改一个任务后下游全乱（可故意制造混乱）
2. **镜3 级联演示**：GanttLens 里拖 `M-2026` 某任务，展示总工期锁死 + 阶段条带同步——操作放慢 0.5x
3. **镜4 代号+悬停**：依次悬停 M-2026 / DC-2026 / OFC-2026，确认只亮当前项目
4. **镜5 文件夹扫描**：`scan-config.json` 配好后触发扫描，展示从空到满的过程
5. **镜6 实时同步**：改一个文件名日期→甘特图自动更新（chokidar 监听）

## 八、拍摄 checklist

- [ ] OBS 1080p 60fps 录屏，鼠标移动放慢
- [ ] 先录口播音频（情绪稳、好剪辑），再对画面
- [ ] 剪映自动字幕 + 手动校对，关键词加粗
- [ ] BGM 低音量电子节奏，55s 卡点收尾
- [ ] 9:16 竖屏，录屏居中，上下留字幕空间
- [ ] 抖音发布带话题 #初赛Demo盲盒 + 活动要求话题

## 九、技术参数

- **录屏工具**：OBS Studio，1080p 60fps
- **剪辑工具**：剪映（自动字幕 + 手动校对）
- **音频**：独立录制口播，后期对画面
- **BGM**：低音量电子节奏型，不盖口播，55s 处卡点收尾
- **导出**：H.264，9:16 竖屏，码率 8-12 Mbps
